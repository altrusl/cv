# Actions and Mutations

Actions and Mutations are needed for changing the Vuex store state ( = managing the databases and modifying documents in them)

# Common use of action

this.$store
    .dispatch('ACTION_NAME', 'params')
    .then(() => {
        console.log("Action has been done");
    });

// or if you don't need to wait for the action completion

this.$store
    .dispatch('ACTION_NAME', 'params');

# Defined Actions

Common argument for an action has the next structure:

{
    db,  // database name
    doc, // document to insert/delete/modify. `_id` parameter denotes the key
    sync // optional, default value is `true`. Indicates whether 
         // to do database synchronization (local <-> remote) after this action
}

All actions with sync = false parameter affect only local databases.

sync = false might be useful if you want to make modifications in a database in a bunch and then do just one synchronization

Currently implemented actions:

  • GET_DOC
  • DELETE_DOC
  • INSERT_DOC
  • UPDATE_OR_INSERT_DOC
  • CREATE_DATABASE
  • DELETE_DATABASE
  • SYNC_DATABASE

# Examples of usage

# GET_DOC

Fetches document with a given _id from the db database

this.$store
    .dispatch("GET_DOC", {
        db: "schools", 
        doc: {
            _id: school.id
        }
    })
    .then((doc) => {
        console.log("Document fetched: " + doc);
    });

# UPDATE_OR_INSERT_DOC

Updates of inserts document into the db database If doc has _id property then this function tries to find this document in the database and update it

If such document has not been found or there is no _id property in doc than a new doc document is inserted into the db database

this.$store
    .dispatch("UPDATE_OR_INSERT_DOC", {
        db: "schools", 
        doc: {
            "title": "new superschool"
        },
        sync: false
    })
    .then(() => {
        console.log("Document has been inserted");
    });

# DELETE_DOC

Deletes document with _id property from the db database

this.$store
    .dispatch("DELETE_DOC", {
        db: "schools", 
        doc: {_
            _id: school.id
        }
    })
    .then(() => {
        console.log("Document has been deleted");
    });

# SYNC_DATABASE

Synchronizes local and remote databases. The synchronization is not live (one time only)

this.$store
    .dispatch('SYNC_DATABASE', 'schools')
    .then(() => {
        console.log("'schools' database was synced");
    });

// or just

this.$store
    .dispatch('SYNC_DATABASE', 'schools');
