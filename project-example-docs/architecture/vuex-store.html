# Data store

All the database data is cached in the Vuex store. It contains the next variables:

  • user - currently logged in user
  • users - database users documents
  • schools - database schools documents
  • curriculums - database curricullums documents
  • lessons - database lessons documents

All the work with database data in the code must be done via Vuex store from now.

# No Vue component should access the database data directly!

# Main advantages of this approach

  • The code becomes synchronous. No more async, await and promises when dealing with data
  • The data becomes reactive. As soon as Vuex store gets updates from databases all the visual elements of Vue components of the system are updated automatically
  • The code becomes shorter, cleaner, more neat and understandable

# How to access data

import { mapState } from "vuex";

// ...

computed: {
    ...mapState(['curriculums']),
},
// now this.curriculums points to an array of all curriculums from the 'curriculums' database

// print out the first curriculum
console.log(this.curriculums[0]);

# How to modify data

To change or delete some document it is required to use Vuex actions

this.$store.dispatch("deleteDoc", {
    db: "schools", 
    id: item._id
}).then(() => {
    console.log("Document deleted successfully");
});

There is more information about data changing in Actions and Mutations

# The transition to a new data model can be carried out gradually

# Realworld example

Transition from direct PouchDB access to Vuex store for Admins list in the User manager

The code before 
async getAdmins() {
    var admins = [];

    Databases.localUsersDB
    .allDocs({ include_docs: true })
    .then((data, p1) => {
        this.adminData = data.rows.forEach(element => {
            if (element.doc.roles[0] === "admin") {
                var item = {
                    adminName: element.doc.name,
                    role: element.doc.roles[0],
                    subadminid: element.doc._id,
                    subadminidrev: element.doc._rev
                };

                admins.push(item);
            } else if (element.doc.roles[0] === "subadmin") {
                var item = {
                    adminName: element.doc.name,
                    role: "LFB", // element.doc.roles[0],
                    subadminid: element.doc._id,
                    subadminidrev: element.doc._rev
                };

                admins.push(item);
            }
        });

        if (this.loginRole === "subadmin") {
            admins = admins.filter(x => x.role === "subadmin");
        }
        this.adminData = admins;
        this.adminData.sort((a,b)=>{
            if (a.adminName.toLowerCase() > b.adminName.toLowerCase()) return 1;
            if (a.adminName.toLowerCase() < b.adminName.toLowerCase()) return -1;
            return 0;
        })
        this.pagesAdmins = Math.ceil(
            this.adminData.length / this.itemPerPage
        );
    });
},

  • Use this.users
  • Change element.doc for element
